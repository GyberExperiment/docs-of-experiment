# **Структура организации Gybernaty: система уровней - UnitTypes, критерии роста и мотивации**  


---

## **1. Общие положения**  
Настоящий документ определяет:  
1. Иерархию участников сообщества Gybernaty.  
2. Критерии перехода между уровнями.  
3. Систему вознаграждений и управления.  
4. Обязательные процедуры контроля качества.  


---

## **2. Уровни участников - Unit Types**

### **2.1. Unit (Начинающий)**
**Описание**: Базовый уровень для новых участников, осваивающих фундаментальные навыки разработки и принципы работы децентрализованных систем и прочих технологий актуальных областей.

**Требования**:
- Регистрация в системе Gybernaty ID
- Прохождение обязательных курсов:
  - «Основы Git и GitHub» (минимум 80% баллов)
  - «Блокчейн для начинающих» (сертификационный экзамен)
  - «Основы командной строки» (Linux/MacOS/Windows)

**Обязанности**:
- Выполнение 3+ учебных задач ежемесячно:
  - Тестирование MVP-проектов
  - Написание технической документации
  - Участие в код-ревью (наблюдатель)
- Посещение 2+ обучающих вебинаров в месяц

**Привилегии**:
- Доступ к GyberUniversity (базовые курсы)
- Участие в закрытом чате новичков
- Возможность подавать заявки на стажировки в проектах
- Назначение ментора из числа Dev+

**Срок пребывания**: 1-6 месяцев (в зависимости от начального уровня и прогресса)

---

### **2.2. Dev (Разработчик)**
**Описание**: Основной производственный состав, выполняющий ключевые технические задачи по специализациям.

**Требования**:
- Успешное завершение уровня Unit или наличие навыков уровня Dev
- 5+ одобренных PR в основных репозиториях
- Защита проекта уровня Junior:
  - Фронтенд: Разработка компонента на React/Next.js
  - Бэкенд: REST API на выбранном стеке
  - Блокчейн: Простой смарт-контракт с тестами

**Обязанности**:
- Реализация 3+ задач ежемесячно по специализации
- Участие в планировании спринтов
- Проведение код-ревью для Unit
- Поддержка документации проектов

**Привилегии**:
- Доступ к проектам с бюджетом до 10 B Gbr
- Право голоса в DAO (вес 1x)
- Доступ к промежуточным курсам GyberUniversity Pro
- Возможность стать ментором

**Специализации**:
1. Frontend (React/Next.js + Web3)
2. Backend (Python/Node.js/Go/Rust)
3. Blockchain (Solidity/Rust)
4. DevOps (Docker/K8s/CI-CD)

---

### **2.3. LeadDev (Ведущий разработчик)**
**Описание**: Технические лидеры, отвечающие за реализацию комплексных проектов и наставничество.

**Требования**:
- 3-6 месяцев в статусе Dev или 1 месяц при наличии навыков уровня LeadDev
- Успешное завершение 1+ проектов среднего уровня:
  - Бюджет 10 B-30 B Gbr
  - Срок реализации 3+ месяца
### **Актуализированные требования к экспертизе по специализациям**  

---

 **1. Frontend-экспертиза**  
**Обязательные навыки:**  
- Оптимизация Core Web Vitals:  
  - LCP < 1.2s через SSR/ISR (Next.js)  
  - CLS < 0.1 с динамическими лейаутами  
  - Интеграция Web Workers для тяжелых вычислений  
- Продвинутый state-менеджмент:  
  - Zustand + Immer для сложных стейтов  
  - Оптимизация рендеров через React Forget (ожидается в React 19)  
- Web3-стек:  
  - Wagmi + Viem для взаимодействия с блокчейном  
  - Адаптация под WalletConnect v2  

**Дополнительные компетенции:**  
- WebAssembly:  
  - Сборка Rust-модулей в WASM для фронтенда  
  - Оптимизация WASM-бандлов (wasm-pack + wasm-opt)  
- Микрофронтенды:  
  - Module Federation (Webpack 5)  
  - Cross-микрофронтендная коммуникация  

---

 **2. Backend-экспертиза**  
**По стекам:**  

 **Python (FastAPI/Starlette)**  
- Асинхронные БД:  
  - SQLAlchemy 2.0 + asyncpg  
  - Оптимизация сложных JOIN через Materialized Views  
- Highload-паттерны:  
  - Rate limiting на уровне Nginx + Lua  
  - Кеширование через RedisJSON  

 **Node.js (NestJS)**  
- Микросервисная архитектура:  
  - gRPC-транспорт с protobuf-сериализацией  
  - Saga-паттерн для распределенных транзакций  
- Безопасность:  
  - JWT-авторизация с OPA/Rego  

 **Golang**  
- Профилирование:  
  - pprof + flamegraph  
  - Аллокация памяти через sync.Pool  
- Cloud-Native:  
  - Автоскейлинг через K8s Operators  

 **Rust**  
- Безопасность:  
  - fuzzing-тесты с cargo-fuzz  
  - Аудит unsafe-блоков  
- Производительность:  
  - SIMD-оптимизации (std::simd)  

---

 **3. Blockchain-экспертиза**  
**Базовый уровень:**  
- EVM-сетевые нюансы:  
  - Gas оптимизация (EIP-4844 для blobs)  
  - MEV-резистентность (Flashbots Protect)  
- Инструменты:  
  - Foundry + Anvil для локального тестирования  
  - Hardhat для сложных деплойментов  

**Продвинутый уровень:**  
- ZK-технологии:  
  - Circom для кастомных схем  
  - Plonky2 для рекурсивных доказательств  
- Кроссчейн:  
  - CCIP (Chainlink) для arbitrary messaging  
  - IBC (Cosmos) для межхаб-коммуникации  

**Аудит контрактов:**  
- Статический анализ:  
  - Slither с кастомными детекторами  
  - Semgrep для поиска шаблонных уязвимостей  
- Динамический анализ:  
  - Echidna для property-based тестов  
  - Differential fuzzing  

---

 **4. DevOps/SRE-экспертиза**  
**Для блокчейн-инфраструктуры:**  
- Ноды:  
  - Оптимизация Geth (снапшоты + state pruning)  
  - Мониторинг через Prometheus + Grafana Loki  
- Безопасность:  
  - Falco для детекции аномалий  
  - Vault для управления приватными ключами  

**Облачные решения:**  
- GitOps:  
  - ArgoCD + Kustomize  
  - Crossplane для управления мультиклаудом  
- Cost-оптимизация:  
  - Spot instances с graceful degradation

### **Критерии подтверждения экспертизы**

 **1. Практический экзамен**
Для каждого уровня и специализации:

**Frontend:**
- Реализация SSR-приложения с:
  - LCP < 1.2s на мобильных устройствах (Lighthouse Audit)
  - Интеграцией Web3-кошелька через WalletConnect v2
  - Оптимизацией бандлов (размер main.js < 150kb)

**Backend:**
- Разработка высоконагруженного API:
  - Обработка 10k+ RPS (тест через k6)
  - Реализация graceful degradation при 90% нагрузке
  - JWT-авторизация с refresh-токенами

**Blockchain:**
- Создание и аудит смарт-контракта:
  - Gas оптимизация (экономия ≥30% от baseline)
  - Защита от reentrancy и frontrunning
  - Полное покрытие тестами (Foundry)

 **2. Код-ревью**
Требования к анализу:

- Для Dev:
  - Проверка 3+ PR средней сложности
  - Выявление минимум 2 критических issues в каждом
  - Предложение оптимизаций

- Для LeadDev+:
  - Аудит архитектурных решений
  - Проверка security best practices
  - Анализ производительности (профайлинг)

---

**Обязанности**:
- Руководство командой 3-5 Dev/Unit
- Проектирование архитектуры решений
- Техническое интервьюирование кандидатов
- Участие в стратегическом планировании

**Привилегии**:
- Доступ к проектам с бюджетом до 50,000 Gbr
- Вес голоса в DAO 3x
- Доступ к премиум-курсам
- 2% от бюджета управляемых проектов
- Право инициировать новые проекты

### **2.4. ArchDev (Архитектор)**
**Описание**: Высший технический уровень, определяющий стратегию развития всей экосистемы.

**Требования**:
- Реализация 1+ enterprise-проекта:
  - Бюджет 50 B+ Gbr
  - Кросс-функциональная команда
  - Интеграция с внешними системами
- Вклад в ядро Gybernaty:
  - Разработка стандартов
  - Системные улучшения

**Обязанности**:
- Проектирование системной архитектуры
- Техническое лидерство в направлении
- Переговоры с ключевыми партнерами
- Формирование технологического стека
- Арбитраж технических споров

**Привилегии**:
- Неограниченный доступ ко всем проектам
- Вес голоса в DAO 5x
- 5% от доходов управляемого направления
- Право вето на критические решения
- Доступ к закрытым встречам с инвесторами

**Процедура назначения**:
1. Выдвижение 2+ действующими ArchDev
2. Публичная защита портфолио
3. Тайное голосование (70% одобрения)
4. Инаугурация с передачей NFT-сертификата

## **3. Критерии перехода между уровнями**  
### **3.1. Unit → Dev**  
1. Завершение курсов:  
   - Git & CI/CD.  
   - Основы Solidity.  
2. Практические задачи:  
   - 5 успешных PR (багфиксы, тесты).  
3. Ревью кода от LeadDev+.  

### **3.2. Dev → LeadDev**  
1. Реализация 2+ проектов уровня «Medium».  
2. Навыки:  
   - Rust или Go (подтвержденные тестом).  
   - Аудит смарт-контрактов.  
3. Положительные отзывы от 5+ участников.  

*(Подробные условия для всех уровней — см. Приложение 2.)*

## **4. Система вознаграждений**  
### **4.1. Начисление токенов Gbr**  
| **Действие**               | Unit | Dev  | LeadDev | ArchDev |  
|----------------------------|------|------|---------|---------|  
| Завершение задачи          | 10   | 30   | 50      | –       |  
| Управление проектом        | –    | 100  | 200 + 2% бюджета | 500 + 5% бюджета |  

### **4.2. Дополнительные механизмы**  
- **Стейкинг**:  
  - LeadDev/ArchDev повышают вес голоса до 1.5x при стейке от 500 Gbr.  
- **Сжигание токенов**:  
  - 10% от доходов проектов сжигается ежеквартально.  

*(Полная экономическая модель — см. Приложение 3.)*

## **5. Управление (DAO)**  
### **5.1. Типы решений**  
1. **Операционные** (LeadDev): Выбор инструментов, дизайн.  
2. **Стратегические** (ArchDev): Бюджет, стек технологий.  
3. **Экосистемные** (Core/Коллективные решения через DAO): Партнерства, сжигание токенов.  

### **5.2. Процедура голосования**  
1. Инициатива → Обсуждение (7 дней) → Голосование (3 дня).  
2. Кворум:  
   - 30% для операционных решений.  
   - 50% для стратегических.

## **6. Контроль качества**  
### **6.1. Система аудита активности и репутации**

#### **Многоуровневая система мониторинга**
**1. Еженедельный микро-аудит:**
- Автоматический трекинг через:
  - GitHub Activity Score (коммиты, PR, ревью)
  - DAO Participation Index (голосования, предложения)
  - Mentorship Points (активность менторства)
- **Пороговые значения:**
  - Dev: ≥15 ед. активности
  - LeadDev: ≥25 ед. активности
  - ArchDev: ≥10 ед. (но с фокусом на стратегические задачи)

**2. Квартальный комплексный аудит:**
- **Технический вклад:**
  - Code Impact Score (по данным GitPrime):
    - Уникальность кода
    - Критичность изменений
    - Влияние на архитектуру
- **Социальный капитал:**
  - Peer Review Rating (оценки коллег)
  - Community Help Score (помощь в чатах)

#### **Динамическая система санкций**
**Гибкая система реагирования:**
1. **Первое нарушение:**
   - Автоматическое назначение куратора
   - Обязательный курс "Тайм-менеджмент в распределенных командах"

2. **Повторное нарушение:**
   - Временное ограничение прав:
     - Dev: запрет на участие в новых проектах (2 недели)
     - LeadDev: понижение веса голоса до 1x
   - Требование пройти стресс-тест (решение сложной задачи за 48 часов)

3. **Хроническое несоответствие:**
   - Автоматический демотейшн с "карантином":
     - LeadDev → Dev: минимум 3 месяца до повторной аттестации
     - Dev → Unit: полный пересмотр квалификации

#### **Система бонусов за стабильность**
- **Ежеквартальные премии:**
  - 100 Gbr за 12 недель 100% активности
  - NFT-бейдж "Несокрушимый" (дает +5% к наградам)
- **Компенсационные механизмы:**
  - Возможность "откупа" пропусков (1 день = 10 Gbr в общий фонд)
  - Система "активностного кредита" (накопление сверхнормы)

#### **Техническая реализация**
1. **Автоматизированная система:**
   - Бот @GyberAuditor в Discord
   - Персональный дашборд с:
     - Прогнозом риска понижения
     - Сравнением с коллегами того же уровня
     - Рекомендациями по улучшению

2. **Человеческий фактор:**
   - Рандомизированная проверка 5% аудитов комитетом ArchDev
   - Возможность апелляции через DAO (стоимость - 50 Gbr)

#### **Особые условия**
- **Для ArchDev:**
  - Альтернативная система оценки:
    - Влияние на стратегию (количество принятых предложений)
    - Экспертная оценка 3 случайных LeadDev
- **Льготные периоды:**
  - Возможность заморозки аудита (максимум 2 недели в год)
  - Сезонные коэффициенты (лето = ×0.8 требований)

*(Полная математическая модель расчетов - см. Приложение 7. Алгоритмы машинного обучения для оценки вклада)*
